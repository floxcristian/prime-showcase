# Architecture Decision Records (ADRs)

This directory contains architecture decision records for prime-showcase, documenting significant technical decisions with context, alternatives, and outcomes.

## Format

MADR-inspired. Each ADR:
- Declares a decision.
- Documents the problem, the root cause (empirically verified when relevant), alternatives considered, and the chosen path.
- Maintains a changelog for ongoing adjustments.

## Adding a new ADR

1. Copy `001-*` as a template.
2. Number sequentially (002, 003, ...).
3. Add to the index above.
4. Link from `CLAUDE.md` if the decision changes day-to-day authoring.
