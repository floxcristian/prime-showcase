# Z-Index

Ocho tokens semánticos, sin números crudos. Espejo de la escala de Primer.

```text
0       base                                  ← default, sin z explícito
10      sticky          (Tailwind: z-10)      ← sticky headers in-page en scroll multi-panel
1000    dropdown                              ← default de Aura para overlays internos
1100    overlay         (p-popover, p-menu)   ← popups anclados a un trigger
1200    drawer          (p-drawer)            ← side sheets en mobile
1300    modal           (p-dialog, confirm)   ← bloquea la interacción de la página
1400    toast           (p-toast)             ← sobre el modal — errores mid-flow quedan visibles
1500    tooltip                               ← siempre arriba, nunca tapado
```

## Reglas

- **Sticky in-page**: `class="sticky top-0 z-10"` — único lugar donde el
  número crudo `z-10` se usa directamente.
- **Overlays de PrimeNG**: el componente maneja su z-index internamente vía
  el preset de Aura. NO sobreescribir con `styleClass`.
- **¿Se necesita un nivel nuevo?** Agregar una entrada nombrada a `zIndex` en el front matter
  de `DESIGN.md` Y a `app.preset.ts` como design token semántico. Nunca usar
  un número crudo en un template.

## Por qué el toast queda sobre el modal

Flujos del mundo real: el usuario abre un confirm dialog (modal), confirma, el
servidor devuelve un error. El toast debe ser visible SIN cerrar el
modal para que el usuario pueda decidir qué hacer. De ahí `toast = 1400 >
modal = 1300`.

## Uso verificado

- `class="sticky top-0 z-10 bg-surface-0 dark:bg-surface-950"` —
  sticky headers de multi-panel + chat / inbox / customers.
- `p-toast` — aparece sobre cualquier `p-dialog` abierto en runtime de producción.
