# Tokens de color

Cada color de la app viene de una de tres capas, en orden de preferencia:

1. **Aura semantic tokens** (`text-color`, `text-muted-color`, `bg-surface-*`,
   `bg-primary`, …). Usar estos para **todo lo de UI**.
2. **Overrides del proyecto** declarados en `src/app/app.preset.ts`:
   `formField.invalidBorderColor: rose.500`, `text.muted.color: surface.600`
   (light) / `surface.300` (dark). Validados con drift detection contra `DESIGN.md`
   por `tools/design-tokens/sync.mjs` en cada `npm run lint`.
3. **Excepciones semánticas** — colores nombrados de Tailwind con significado FIJO:
   `green-500` (dot online / activo), `yellow-500` (BTC), `violet-100` /
   `orange-100` (status badges). Nunca se usan como paleta general de UI.

## Paleta primary

Single-hue 204° (Pantone "Implementos blue"). Derivada del logo de marca
`public/implementos-logo.svg` (`#006DB6`). Toda la rampa comparte H=204, con
S=100% excepto `.400` (88%, bajado para suavizar el azul eléctrico en dark) y
las tintas naturales `.50`–`.300`.

### Contraste verificado

| Par | Contraste | Nivel |
|---|---:|---|
| `primary.500` sobre `surface.0` | 4.9:1 | WCAG 2.1 AA |
| `primary.400` sobre `surface.950` (dark) | 7.0:1 | AAA |
| `primary.700` sobre `surface.0` (active) | 10.5:1 | AAA |

## Colores prohibidos

- ❌ Colores genéricos de Tailwind: `text-gray-500`, `bg-blue-100`, `text-slate-700`, etc.
- ❌ Literales hex / rgb: `#fff`, `rgb(0, 0, 0)` — excepto dentro de datasets de Chart.js
  donde la legend se construye fuera de la cascade de Angular.
- ❌ Strings `var(--p-…)` en componentes TypeScript que no estén leyendo CSS en runtime.

Enforcement vía:

- `showcase/no-hardcoded-colors` — bloquea genéricos / hex.
- `showcase/no-missing-dark-pair` — cada `bg-surface-*` necesita un `dark:bg-surface-*`.
- `tools/design-tokens/sync.mjs` — protege `DESIGN.md` contra drift vs el preset.

Ver `Primitives/Button` y `Primitives/Tag` para uso de tokens en contexto de componente.
